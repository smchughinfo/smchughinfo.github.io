Â Â As described above, the anomaly type fuzzing detection method described in Patent Document 1 has a problem that erroneous detection occurs when information handled by the embedded information processing device changes.

The antenna 11 outputs a message received from the outside to the receiving unit 201. The receiving unit 201 outputs the input message to the fuzzing detection unit 202 (S101).

Â Â The fuzzing detection unit 202 compares the received message with the fuzzing detection database 203, and determines whether the message is for vulnerability search (S102).

Â Â In this embodiment, a white list is used as a method for determining the received message.

For example, the state of the vehicle, the message ID that may be sent, the range of the data portion of the message that can exist with the message ID, and the like are associated and described in the fuzzing detection database 203 in advance.
If the received message does not satisfy these conditions, the fuzzing detection unit 202 determines that the message is for the purpose of searching for a vulnerability (S102: YES).

As teacher data for these machine learning methods, an operation log of a test performed during development can be used.
