Â Â Both enterprise and consumer broadband networks have seen significant and constant increases in traffic volume over the past 3-5 years. Â Â Broadband network operators control their network costs, including increasing network scale, to meet network congestion, reduced application performance and the need to continually upgrade their networks caused by broadband growth, A variety of network optimization solutions and services have been introduced with the aim of improving the performance and security of Internet applications and creating new revenue streams for operators.

Â Â Network-based optimization common solution architecture is a network optimization platform that resides in the data path and is deployed with network elements that redirect traffic to the network optimization platform (eg, routing, switching or dedicated DPI equipment) including.

Â Â Network elements typically employ selective redirection of network traffic to match the type of traffic flow to the network optimization service used.

Â Â Network optimization services typically use an application proxy architecture. A connection is established between two endpoints 'A' and 'B', otherwise (eg Internet browser and web browser), terminated by a proxy 'P', and two separate transport sessions (TCP Or UDP) is formed on the one hand between A and P and on the other hand between P and B. Following connection setup, the proxy P relays data between two sessions at the application level.

Â Â The proxy architecture maintains a transport (TCP or UDP) stack for all sessions flowing between networks, replicates data to relay all data at the application level, and back and forth from the data frame to the application buffer. Has a significant performance penalty due to the need to perform the conversion.

Â Â As a result of these limitations, the proxy architecture limits the throughput of network optimization applications to 1-2 Gbps per standard Intel-based server and simultaneously limits the number of flows supported to thousands. Performance limitations effectively prevent network optimization solutions from scaling to 10/40 / 100Gbps network scale in an economical manner.

Â Â Therefore, an alternative architecture is needed for a network optimization platform that removes the previous bottleneck of the application proxy architecture.
