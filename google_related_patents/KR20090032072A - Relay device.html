In the OSI reference model, the lowest layer (first layer) is the physical layer, the second layer is the data link layer, the third layer is the network layer, the fourth layer is the transport layer, the fifth layer is the session layer, and the sixth layer is the presentation layer. First, the outline of the OSI reference model will be described. The physical layer of the first layer is a layer defining physical electrical characteristics of a signal line, a modulation method of a code, and the like. However, this layer alone is rarely defined and implemented, and is usually defined as, for example, an Ethernet standard and the like together with the data link layer of the second layer.

The data link layer of the second layer is a layer defining data packetization, physical node address, packet transmission / reception method, and the like. This layer defines a protocol for exchanging packets between two nodes via a physical communication medium, assigns an address to each node, specifies the destination of the packet based on the address, and specifies the packet. Send on the communication medium.

As a communication medium, there exist various things, such as copper wiring, wireless, and an optical fiber. In addition, there are many types of connection forms (topologies), such as bus-type, star-type, and ring-type as well as one-to-one opposing connections. The packet transmitted on the communication medium is received by the node when it arrives at the receiving node and passed to the upper protocol layer.

The NIC (Network Interface Card) driver disposed through the physical layer and the data link layer is an expansion board for connecting a PC or a printer to a local area network (LAN). Often just a network card is on the ethernet.

This NIC driver allows a node (device) to transmit data to monitor the cable vacancy and to start transmission when the cable is empty. At this time, if a plurality of nodes start transmitting at the same time, data collides and is destroyed in the cable, so that both of them stop transmitting and wait for a random time to resume transmission. This allows a plurality of nodes to share a cable and communicate with each other.

The network layer of the third layer is a layer that defines a communication method between any two nodes. Speaking of TCP / IP, it corresponds to the IP layer. In the data link layer, although communication can be performed between nodes on the same network medium, it is this network that uses the function to communicate while routing between any two nodes existing on the network. It is the role of the floor.

Here, routing means selecting and transmitting an optimal path when transmitting a packet to a target host in a TCP / IP network. For example, in Ethernet, only nodes on the same segment can communicate with each other, but in the network layer, communication is performed by routing a packet between two Ethernet segments.

Also, it is possible to route packets regardless of the physical network medium, such as routing to a dial-up Point-to-Point Protocol (PPP) line that connects a computer to a network (Ethernet) through a telephone line, or to a dedicated line using optical fiber. Can be rooted. For this purpose, an address (IP address, in case of TCP / IP) that does not depend on a physical medium is usually assigned to each node, and routing is performed based on this.

IPsec encrypts all communication transmitted from the host at this network layer, i.e., at the IP level, so that the user can enable secure communication without being aware of the application.

The transport layer of the fourth layer is a layer that provides a function for communicating between two processes running on each node, and is a protocol layer. Speaking of TCP / IP, TCP is equivalent to this. In addition, although the network layer provides a function for communicating between two nodes, it is the role of TCP to provide a virtual communication path without error between two processes (applications) using this.

In other words, data can be sent at the network layer, but there is no guarantee that the data will reach the other party. There is also no guarantee that the data will arrive correctly in the order in which they were sent. Thus, to make it easy to use in an application, it is TCP that provides an error free communication path. If necessary, the data is resent and recovered.

In addition to TCP, UDP is also provided in this transport layer. However, the difference between UDP and TCP is that the protocol is the data-compensated protocol, which is slower than the UDP.
