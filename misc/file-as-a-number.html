A file as a number...

Unicode just maps characters to numbers. For example, A is represented by the number 65, B is represented by the number 66, etc. For a complete list checkout this chart.
Here we just take each Unicode number and convert it into a binary number. For example, 65 in binary is 1000001, 66 in binary is 1000010, etc. So the string "AB", when encoded as ASCII, has a binary representation of 1000001 1000010.
If we string 1000001 and 1000010 together we get 10000011000010 which is the binary representation of 8386. Neat, so "AB" == 65 66 == 1000001 1000010 == 8386.

foo.txt is just a regular text file. This same concept applies to other file types because no matter what type of file you are dealing with it ultimately gets saved as 1's and 0's. Text files are just an easy example to demonstrate. But .png, .jpg, .mp3, mp4, .exe, etc. all get saved in binary just like foo.txt.
